웹 플랫폼 백엔드 아키텍처 개편

NestJS AWS ECS MySQL 8.0 Express

개요

Express + SST 기반 모놀리식 서버를 NestJS로 마이그레이션하고, ECS 기반 컨테이너 배포 환경을 구축했습니다. 코드 컨벤션과 모듈 구조를 표준화하고, 개발 착수 전 아키텍처 공유 및 리뷰 프로세스를 도입했습니다.

배경

기존 시스템은 Express + SST 기반의 모놀리식 구조였습니다. 코드 컨벤션 없이 개발자마다 다른 패턴으로 작성되어 유지보수가 어려웠고, 도메인 간 경계가 모호하여 변경 사항이 예상치 못한 곳에 영향을 주는 일이 빈번했습니다.

설계 및 구현

NestJS 마이그레이션 아키텍처

  • 도메인별 모듈 분리: 회원, 주문, 상품, 결제 등 핵심 도메인 독립 모듈화
  • 코드 컨벤션 및 모듈 구조 표준화
  • 개발 착수 전 아키텍처 공유 및 리뷰 프로세스 도입

ECS 기반 배포 환경

  • AWS ECS 컨테이너 배포 환경 구축
  • 유휴 리소스 활용으로 QA 서버를 ECS로 분리/증설
  • 병렬 배포 구조 도입으로 배포 병목 해소

주요 성과

  • 핵심 도메인 API 개발 (회원/주문/상품/결제)
  • 코드 중복 감소 및 일관된 개발 패턴 확립
  • QA 환경 병렬 배포로 배포 대기시간 제거
  • ISMS-P 인증 심사 대응 및 보안 조치 수행
